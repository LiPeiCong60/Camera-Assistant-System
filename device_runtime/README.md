# device_runtime

树莓派运行时模块，技术栈按定稿使用 Python，并优先复用 `demo` 的已有核心逻辑。

当前阶段只建立目录边界，后续按最小改动迁移 `demo` 中的检测、跟踪、云台、模板与运行时服务。

- `api`：树莓派本地 HTTP 接口
- `vision`：检测、视频输入、视觉处理
- `control`：云台控制与跟踪控制
- `templates`：模板数据与模板构图能力
- `services`：运行时服务编排

明确不迁移 Tkinter GUI。

## 当前硬件说明

- 树莓派真实硬件统一使用 TTL 总线舵机
- 舵机必须使用独立外接电源，并与树莓派控制侧共地
- `PCA9685` 方案已废弃，不再作为当前仓库的设备端控制方案
- 本地 Windows 联调默认仍可使用 `mock` 驱动模拟云台
