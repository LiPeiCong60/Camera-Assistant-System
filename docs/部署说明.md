# 部署说明

本文给出本机开发、真机联调和树莓派运行的推荐方式。

## 本机开发

### 1. 启动 PostgreSQL

创建数据库：

```powershell
createdb camera_assistant
```

### 2. 启动后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DATABASE_URL="postgresql+psycopg://postgres:postgres@127.0.0.1:5432/camera_assistant"
$env:BACKEND_AUTH_SECRET="change-this-in-real-env"

python init_db.py
uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. 启动设备运行时

Windows 本机联调用 mock 云台：

```powershell
cd device_runtime
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DEVICE_SERVO_DRIVER="mock"
uvicorn device_runtime.api.app:app --reload --host 0.0.0.0 --port 8001
```

### 4. 启动管理后台

```powershell
cd admin_web
npm install
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
npm run dev
```

### 5. 启动手机端

Android 模拟器：

```powershell
cd mobile_client
flutter pub get
flutter run `
  --dart-define=API_BASE_URL=http://10.0.2.2:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://10.0.2.2:8001
```

真机：

```powershell
flutter run `
  --dart-define=API_BASE_URL=http://192.168.1.20:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://192.168.1.30:8001
```

## 树莓派部署

```bash
cd device_runtime
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

export DEVICE_RPI_PROFILE=performance
export DEVICE_SERVO_DRIVER=ttl_bus
export DEVICE_TTL_SERIAL_PORT=/dev/ttyUSB0
export DEVICE_PAN_SERVO_ID=0
export DEVICE_TILT_SERVO_ID=1

uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

现场建议：

- 手机和树莓派使用同一 5GHz Wi-Fi。
- 网络拥挤时让树莓派开热点，手机直连热点。
- 真机 App 中设备运行时地址填写树莓派 IP，例如 `http://192.168.1.30:8001`。
- 不要在真机上填写 `127.0.0.1`。

## 生产/演示安全项

- `BACKEND_AUTH_SECRET` 必须使用真实随机密钥，不要使用文档示例值。
- AI Provider API key 只在管理后台/数据库中维护，不要写入移动端代码。
- `.env`、`uploads`、`captures`、模型文件、日志、数据库文件不要提交到 Git。
- 管理后台生产部署时应配置 HTTPS、反向代理和更严格 CORS。

## 重启规则

| 修改内容 | 是否需要重启 |
| --- | --- |
| `device_runtime` Python 代码 | 需要重启设备端 `uvicorn` |
| 树莓派环境变量、串口、舵机 ID | 需要重启设备端 `uvicorn` |
| Flutter 代码 | 需要重新运行或重装 App |
| Android 权限、Gradle、依赖 | 需要重新构建 APK |
| 后端 Python 代码 | 开发 reload 可自动加载；生产需要重启 |
| 管理后台代码 | 开发 Vite 自动刷新；生产需要重新构建 |

## APK 构建

```powershell
cd mobile_client
flutter clean
flutter pub get
flutter build apk --release `
  --dart-define=API_BASE_URL=http://192.168.1.20:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://192.168.1.30:8001
```

输出：

```text
mobile_client/build/app/outputs/flutter-apk/app-release.apk
```
