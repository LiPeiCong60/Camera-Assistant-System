# 部署说明

## 1. 环境准备

### 必需软件

- Python 3.12
- PostgreSQL
- Node.js 20+
- Flutter SDK
- Android Studio（如需运行手机端）

### 项目目录

- 根目录：`E:\BianChengWenJian\Camera Assistant`

## 2. Python 虚拟环境

项目已使用根目录虚拟环境：

```powershell
cd "E:\BianChengWenJian\Camera Assistant"
.\.venv\Scripts\Activate.ps1
```

## 3. PostgreSQL

### 数据库连接

默认开发连接示例：

```powershell
$env:DATABASE_URL="postgresql+psycopg://postgres:<your_password>@127.0.0.1:5432/camera_assistant"
```

### 初始化数据库

```powershell
.\.venv\Scripts\python.exe -m backend.init_db
```

## 4. 启动 backend

```powershell
cd "E:\BianChengWenJian\Camera Assistant"
.\.venv\Scripts\Activate.ps1
$env:DATABASE_URL="postgresql+psycopg://postgres:<your_password>@127.0.0.1:5432/camera_assistant"
python -m backend.main
```

默认地址：

- `http://127.0.0.1:8000`
- 健康检查：`http://127.0.0.1:8000/api/health`

## 5. 启动 device_runtime

```powershell
cd "E:\BianChengWenJian\Camera Assistant"
.\.venv\Scripts\Activate.ps1
uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

默认地址：

- `http://127.0.0.1:8001`
- 健康检查：`http://127.0.0.1:8001/api/device/health`

## 6. 启动 admin_web

```powershell
cd "E:\BianChengWenJian\Camera Assistant\admin_web"
npm install
npm run dev
```

默认地址：

- `http://127.0.0.1:5173`

## 7. 启动 mobile_client

### 先拉依赖

```powershell
cd "E:\BianChengWenJian\Camera Assistant\mobile_client"
flutter pub get
```

### Android 模拟器运行

```powershell
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000/api --dart-define=DEVICE_API_BASE_URL=http://10.0.2.2:8001
```

### 真机运行

将下面 IP 换成你电脑的局域网地址：

```powershell
flutter run --dart-define=API_BASE_URL=http://192.168.1.100:8000/api --dart-define=DEVICE_API_BASE_URL=http://192.168.1.100:8001
```

## 9. AI Provider 配置建议

当前后台已支持多厂商、多模型、多密钥配置。

如使用 LongCat（OpenAI 兼容）：

- 厂商：`LongCat`
- 兼容格式：`OpenAI 兼容`
- Base URL：`https://api.longcat.chat/openai`
- 模型名：`LongCat-Flash-Omni-2603`

注意：

- 不建议把 Base URL 填成完整的 `/v1/chat/completions`
- 应填写 provider 根路径

## 10. 本地静态上传目录

backend 当前使用本地静态存储作为 v1 默认方案：

- 上传目录：`uploads/captures/...`
- 访问路径：`/uploads/...`

适用场景：

- 本地开发
- 联调收口
- 验收演示

后续如需上线生产，可再替换成对象存储。
