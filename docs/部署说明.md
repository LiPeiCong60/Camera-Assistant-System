# 部署说明

## 1. 环境准备

推荐版本：

| 依赖 | 用途 |
| --- | --- |
| Python 3.12 | backend 和 device_runtime |
| PostgreSQL 15+ | 业务数据库 |
| Node.js 20+ | admin_web |
| Flutter SDK 3.35+ | mobile_client |
| Git | 代码管理 |

树莓派端如果接真实云台，需要：

- TTL 总线舵机
- 独立舵机电源
- 树莓派和舵机控制侧共地
- 可用串口，例如 `/dev/ttyUSB0`

## 2. 数据库

创建数据库：

```sql
CREATE DATABASE camera_assistant;
```

后端连接串示例：

```powershell
$env:DATABASE_URL="postgresql+psycopg://postgres:postgres@127.0.0.1:5432/camera_assistant"
```

初始化方式：

```powershell
cd backend
python init_db.py
```

`init_db.py` 会调用 SQLAlchemy 建表，并执行兼容补丁。也可以参考 `database/schema.sql` 手动建表。

## 3. 启动 backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DATABASE_URL="postgresql+psycopg://postgres:postgres@127.0.0.1:5432/camera_assistant"
$env:BACKEND_AUTH_SECRET="change-this-in-real-env"
$env:BACKEND_UPLOADS_DIR="E:\BianChengWenJian\Camera Assistant\uploads"

python init_db.py
uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

检查：

```powershell
curl http://127.0.0.1:8000/api/health
```

## 4. 启动 device_runtime

### 本机 mock 模式

```powershell
cd device_runtime
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DEVICE_SERVO_DRIVER="mock"
uvicorn device_runtime.api.app:app --reload --host 0.0.0.0 --port 8001
```

### 树莓派模式

```bash
cd device_runtime
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

export DEVICE_SERVO_DRIVER=ttl_bus
export DEVICE_TTL_SERIAL_PORT=/dev/ttyUSB0
export DEVICE_PAN_SERVO_ID=0
export DEVICE_TILT_SERVO_ID=1

uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

`device_runtime/requirements.txt` 默认不安装 `ultralytics` 和 `torch`，避免树莓派拉取不合适的 CUDA 包。检测器会按运行环境退化到 MediaPipe 或 OpenCV HOG。

## 5. 启动 admin_web

```powershell
cd admin_web
npm install
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173
```

生产构建：

```powershell
npm run build
```

## 6. 启动 mobile_client

### Android 模拟器

```powershell
cd mobile_client
flutter pub get
flutter run `
  --dart-define=API_BASE_URL=http://10.0.2.2:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://10.0.2.2:8001
```

### 真机

把地址改为电脑局域网 IP：

```powershell
flutter run `
  --dart-define=API_BASE_URL=http://192.168.1.10:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://192.168.1.20:8001
```

如果设备运行时跑在树莓派上，`DEVICE_API_BASE_URL` 应指向树莓派 IP。

## 7. AI Provider 配置

进入管理后台的 AI 配置页，新增配置：

- `provider_code`：唯一编号，例如 `openai_main`
- `vendor_name`：例如 `openai`、`longcat`、`ollama`
- `provider_format`：当前使用 `openai_compatible`
- `api_base_url`：建议填到 chat completions endpoint
- `api_key`：Ollama 可为空，其他 Provider 通常必填
- `model_name`：支持视觉输入的模型
- `enabled`：必须开启
- `is_default`：至少保留一个默认项

套餐页可以把默认 Provider 写入 `feature_flags.default_ai_provider_code`，也可以维护 `available_ai_provider_codes`。

## 8. 上传目录

backend 会挂载静态路径：

```text
/uploads -> BACKEND_UPLOADS_DIR
```

手机上传图片会保存到：

```text
uploads/captures/user_<id>/<yyyy-mm-dd>/<uuid>.<ext>
```

后台推荐模板图片会保存到：

```text
uploads/templates/admin_<id>/<yyyy-mm-dd>/<uuid>.<ext>
```

生产环境建议把上传目录放在持久化磁盘，并由反向代理或对象存储托管。

## 9. 常见问题

### device_runtime/main.py 不能启动

这是正常现状。实际入口是：

```powershell
uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

### Android 模拟器连不上 127.0.0.1

模拟器里的 `127.0.0.1` 指模拟器自身。访问电脑宿主机通常使用 `10.0.2.2`。

### 树莓派安装依赖时出现 torch 或 nvidia 包问题

不要默认安装 `ultralytics`。当前依赖清单已经避免默认拉取。确实需要 YOLO 时，先确认 CPU 版 torch 可导入，再手动安装 `ultralytics --no-deps` 并设置 `DEVICE_ENABLE_YOLO=1`。

### AI 任务失败但接口返回了任务对象

这是设计行为。Provider 未配置、未启用、缺密钥或调用失败时，后端会落库一个 `status=failed` 的 `ai_tasks`，方便后台查看原因。
