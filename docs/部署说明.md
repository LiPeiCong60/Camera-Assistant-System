### 树莓派舵机硬件方案（当前唯一推荐方案）

当前树莓派硬件统一按“总线舵机 + 独立电源”部署：

- 舵机类型：TTL 总线舵机
- 供电方式：舵机使用独立外接电源
- 接线要求：树莓派控制板、总线舵机控制器、舵机电源必须共地
- 当前运行时默认驱动：Linux / 树莓派环境默认走 `ttl_bus`

明确说明：

- `PCA9685` 方案已废弃，不再作为当前项目的树莓派舵机控制方案
- `device_runtime` 现阶段只保留总线舵机串口驱动与本机 `mock` 调试模式
- 如果在 Windows 开发机本地调 UI 或 API，默认仍会使用 `mock`，不会误连真实舵机

### 树莓派启动前建议配置的环境变量

在树莓派上启动 `device_runtime` 前，建议显式配置：

```bash
export DEVICE_SERVO_DRIVER=ttl_bus
export DEVICE_TTL_SERIAL_PORT=/dev/ttyUSB0
export DEVICE_TTL_BAUDRATE=115200
export DEVICE_TTL_MOVE_TIME_MS=120
export DEVICE_PAN_SERVO_ID=0
export DEVICE_TILT_SERVO_ID=1
```

按实际机构继续调整的可选参数：

- `DEVICE_PAN_MIN_ANGLE`
- `DEVICE_PAN_MAX_ANGLE`
- `DEVICE_PAN_HOME_ANGLE`
- `DEVICE_PAN_MAX_STEP_DEG`
- `DEVICE_TILT_MIN_ANGLE`
- `DEVICE_TILT_MAX_ANGLE`
- `DEVICE_TILT_HOME_ANGLE`
- `DEVICE_TILT_MAX_STEP_DEG`

如果只是在开发机本地模拟设备联动，而不是连接真实总线舵机，可临时切回：

```powershell
$env:DEVICE_SERVO_DRIVER="mock"
uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

## 9. AI Provider 配置建议

当前后台已支持多厂商、多模型、多密钥配置。

如使用 LongCat（OpenAI 兼容）：

- 厂商：`LongCat`
- 兼容格式：`OpenAI 兼容`
- Base URL：`https://api.longcat.chat/openai`
- 模型名：`LongCat-Flash-Omni-2603`

注意：

- 不建议把 Base URL 填成完整的 `/v1/chat/completions`
- 应填写 provider 根路径

## 10. 本地静态上传目录

backend 当前使用本地静态存储作为 v1 默认方案：

- 上传目录：`uploads/captures/...`
- 访问路径：`/uploads/...`

适用场景：

- 本地开发
- 联调收口
- 验收演示

后续如需上线生产，可再替换成对象存储。
