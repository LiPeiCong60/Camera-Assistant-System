# 部署说明

本文档说明本地和树莓派联调的启动方式。完整架构说明见 [技术架构与运行机制详解](./技术架构与运行机制详解.md)。

## 1. 环境准备

需要安装：

- Python 3.11 或项目依赖兼容版本。
- PostgreSQL。
- Flutter SDK。
- Android Studio / Android SDK。
- Node.js。
- 树莓派真实设备还需要串口权限和 TTL 总线舵机接线。

建议分别为 `backend` 和 `device_runtime` 建虚拟环境，避免 AI、OpenCV、WebRTC 依赖互相污染。

## 2. 数据库

创建数据库后初始化：

```powershell
cd backend
$env:DATABASE_URL="postgresql+psycopg://postgres:postgres@127.0.0.1:5432/camera_assistant"
python init_db.py
```

也可以手动执行：

```powershell
psql -d camera_assistant -f database/schema.sql
```

## 3. 启动 backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DATABASE_URL="postgresql+psycopg://postgres:postgres@127.0.0.1:5432/camera_assistant"
$env:BACKEND_AUTH_SECRET="change-this-in-real-env"

python init_db.py
uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

健康检查：

```powershell
curl http://127.0.0.1:8000/api/health
```

## 4. 启动 device_runtime

### 本机 mock 模式

```powershell
cd device_runtime
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:DEVICE_SERVO_DRIVER="mock"
uvicorn device_runtime.api.app:app --reload --host 0.0.0.0 --port 8001
```

### 树莓派模式

```bash
cd device_runtime
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

export DEVICE_SERVO_DRIVER=ttl_bus
export DEVICE_TTL_SERIAL_PORT=/dev/ttyUSB0
export DEVICE_PAN_SERVO_ID=0
export DEVICE_TILT_SERVO_ID=1

uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

### WebRTC 依赖

`device_runtime/requirements.txt` 已包含：

- `aiortc`
- `websockets`
- `opencv-python-headless`
- `mediapipe`

新增或更新这些依赖后，必须重新 `pip install -r requirements.txt` 并重启 `device_runtime`。

## 5. 启动 admin_web

```powershell
cd admin_web
npm install
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173
```

## 6. 启动 mobile_client

### Android 模拟器

```powershell
cd mobile_client
flutter pub get
flutter run `
  --dart-define=API_BASE_URL=http://10.0.2.2:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://10.0.2.2:8001
```

### 真机

真机必须使用电脑或树莓派局域网 IP：

```powershell
flutter run `
  --dart-define=API_BASE_URL=http://192.168.1.20:8000/api `
  --dart-define=DEVICE_API_BASE_URL=http://192.168.1.30:8001
```

不要使用：

- `127.0.0.1`：指向手机自己。
- `10.0.2.2`：只适用于 Android 模拟器。

也可以在 App 设置页里保存“业务后台地址”和“设备运行时地址”。WebRTC 会复用设备运行时地址，不需要单独配置。

## 7. Android 权限和 cleartext

设备联动 WebRTC 和局域网 HTTP 依赖以下配置：

- `INTERNET`
- `CAMERA`
- `RECORD_AUDIO`
- `ACCESS_NETWORK_STATE`
- `CHANGE_NETWORK_STATE`
- `MODIFY_AUDIO_SETTINGS`
- `android:usesCleartextTraffic="true"`

改动 Android Manifest、Gradle 或 Flutter 插件后，需要重新构建并安装 APK：

```powershell
cd mobile_client
flutter build apk --debug
```

APK 输出：

```text
mobile_client/build/app/outputs/flutter-apk/app-debug.apk
```

## 8. 设备联动验证

1. 启动 `backend`、`device_runtime`。
2. 手机和设备运行端接入同一局域网。
3. 在手机设置页填入真实局域网 IP。
4. 用手机浏览器访问 `http://设备IP:8001/api/device/health`。
5. 打开设备联动页，使用 `mobile_push`。
6. 正常情况下状态显示 WebRTC 推流，预览区域显示设备端处理后的画面。
7. WebRTC 失败时 App 会尝试 WebSocket/JPEG fallback。

## 9. AI Provider 配置

业务后端 AI Provider 由管理后台维护：

1. 启动 backend 和 admin_web。
2. 登录管理后台。
3. 打开 AI Provider 配置页面。
4. 创建或更新 Provider，填写 base URL、model、API key。
5. 套餐可以通过 `feature_flags` 指定默认 Provider 或可用 Provider。

设备端 AI 编排是本地运行逻辑，主要用于自动找角度和背景锁；它不等同于业务后端 AI 任务。

## 10. 常见问题

### 真机无法连接设备运行时

检查：

- 手机和电脑/树莓派是否在同一 Wi-Fi。
- `device_runtime` 是否用 `--host 0.0.0.0` 启动。
- Windows 防火墙是否放行 `8001`。
- 设置页设备地址是否为局域网 IP。
- 手机浏览器能否打开 `/api/device/health`。

### WebRTC 启动失败

检查：

- 设备端是否安装 `aiortc` 和 `av`。
- 当前 session 是否是 `mobile_push`。
- Android 是否授予摄像头权限。
- 是否重新安装了包含 `flutter_webrtc` 的 APK。
- 局域网是否阻断 UDP/本机候选连接。

### fallback WebSocket 失败

检查设备端是否安装 `websockets`。如果日志出现 `Unsupported upgrade request` 或 `No supported WebSocket library detected`，执行：

```powershell
cd device_runtime
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

然后重启 `device_runtime`。

### 树莓派安装依赖慢或失败

默认依赖没有安装 YOLO 和 torch。不要直接在树莓派上无脑安装 `ultralytics`，否则可能拉取 CUDA 相关包。先使用 MediaPipe/OpenCV 路径验证。

### 修改后需要重启什么

- 改 `device_runtime` 代码或依赖：重启 `uvicorn device_runtime.api.app:app`。
- 改 `backend` 代码或依赖：重启 backend。
- 改 Flutter 依赖、Manifest、Gradle：重新 `flutter pub get` 并重新安装 APK。
- 改管理端依赖：重启 `npm run dev`。
