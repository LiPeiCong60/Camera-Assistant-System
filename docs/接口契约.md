# Camera Assistant 接口契约

## 1. 通用约定

业务后端基础地址默认是 `http://127.0.0.1:8000/api`。设备运行时基础地址默认是 `http://127.0.0.1:8001`。

业务后端响应统一为：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

设备运行时也采用相同 envelope。失败时一般返回 `success=false`，或由 FastAPI 返回 `detail`。

需要登录的业务后端接口使用：

```http
Authorization: Bearer <access_token>
```

## 2. 核心对象

### User

| 字段 | 说明 |
| --- | --- |
| `id` | 用户 ID |
| `user_code` | 用户编号 |
| `phone` / `email` | 登录或联系字段 |
| `display_name` | 显示名 |
| `role` | `user` 或 `admin` |
| `status` | `active`、`inactive`、`disabled` |

### Plan

| 字段 | 说明 |
| --- | --- |
| `plan_code` | 套餐编号 |
| `price_cents` | 价格，单位分 |
| `capture_quota` | 抓拍额度 |
| `ai_task_quota` | AI 任务额度 |
| `feature_flags` | 功能开关，包含 AI Provider 选择 |

### Template

| 字段 | 说明 |
| --- | --- |
| `template_type` | `pose`、`background`、`composition` |
| `source_image_url` | 原图地址 |
| `preview_image_url` | 预览图 |
| `template_data` | 姿势点、归一化框和锚点 |
| `is_recommended_default` | 是否为后台推荐模板 |
| `recommended_sort_order` | 推荐排序 |

### CaptureSession

| 字段 | 说明 |
| --- | --- |
| `session_code` | 会话编号 |
| `mode` | `mobile_only`、`device_link`、`MANUAL`、`AUTO_TRACK`、`SMART_COMPOSE` |
| `device_id` | 可为空 |
| `template_id` | 可为空 |
| `metadata` | 手机端平台、拍摄设置等扩展信息 |

### Capture

| 字段 | 说明 |
| --- | --- |
| `capture_type` | `single`、`photo`、`burst`、`best`、`background` |
| `file_url` | 图片地址 |
| `storage_provider` | `local` 或 `local_static` |
| `is_ai_selected` | 是否被 AI 选优 |
| `score` | AI 或业务评分 |

### AiTask

| 字段 | 说明 |
| --- | --- |
| `task_type` | `analyze_photo`、`background_lock`、`batch_pick` 等 |
| `status` | `pending`、`running`、`succeeded`、`failed`、`cancelled` |
| `provider_name` | 使用的 AI Provider |
| `response_payload` | 原始结构化结果 |
| `recommended_pan_delta` / `recommended_tilt_delta` | 建议云台增量 |
| `target_box_norm` | 背景锁目标框，格式 `[x, y, w, h]` |

## 3. 手机端到业务后端

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/mobile/auth/login` | 手机端登录 |
| `POST` | `/mobile/auth/register` | 手机端注册 |
| `GET` | `/mobile/me` | 当前用户 |
| `GET` | `/mobile/plans` | 套餐列表 |
| `GET` | `/mobile/subscription` | 当前订阅 |
| `GET` | `/mobile/templates` | 可用模板，包含用户模板和推荐模板 |
| `POST` | `/mobile/templates` | 创建模板 |
| `DELETE` | `/mobile/templates/{template_id}` | 删除用户模板 |
| `POST` | `/mobile/sessions` | 创建拍摄会话 |
| `POST` | `/mobile/captures/file` | multipart 上传图片文件 |
| `POST` | `/mobile/captures/upload` | 创建抓拍记录 |
| `POST` | `/mobile/ai/analyze-photo` | 单图构图分析 |
| `POST` | `/mobile/ai/analyze-background` | 背景锁分析 |
| `POST` | `/mobile/ai/batch-pick` | 连拍选优 |
| `GET` | `/mobile/ai/tasks/{task_id}` | 查询 AI 任务 |
| `GET` | `/mobile/history/sessions` | 历史会话 |
| `GET` | `/mobile/history/captures` | 历史抓拍 |

### 创建会话

```json
{
  "device_id": 1,
  "template_id": 2,
  "mode": "mobile_only",
  "metadata": {
    "mobile_platform": "android"
  }
}
```

### 上传图片文件

`POST /mobile/captures/file` 使用 `multipart/form-data`，字段名为 `file`。返回：

```json
{
  "file_url": "http://127.0.0.1:8000/uploads/captures/user_1/2026-04-23/xxx.jpg",
  "storage_provider": "local_static",
  "storage_path": "E:/.../uploads/captures/user_1/2026-04-23/xxx.jpg",
  "relative_path": "captures/user_1/2026-04-23/xxx.jpg",
  "original_filename": "photo.jpg",
  "content_type": "image/jpeg"
}
```

### 创建抓拍记录

```json
{
  "session_id": 10,
  "capture_type": "single",
  "file_url": "http://127.0.0.1:8000/uploads/captures/user_1/xxx.jpg",
  "thumbnail_url": null,
  "width": 1920,
  "height": 1080,
  "storage_provider": "local_static",
  "is_ai_selected": false,
  "score": null,
  "metadata": {
    "relative_path": "captures/user_1/xxx.jpg"
  }
}
```

### AI 分析结果约定

单图分析和背景分析期望 AI 返回：

```json
{
  "task_type": "background_lock",
  "recommended_pan_delta": 2.0,
  "recommended_tilt_delta": -1.0,
  "target_box_norm": [0.38, 0.18, 0.24, 0.66],
  "summary": "建议略微向右调整机位，保留人物上方空间。",
  "score": 91
}
```

连拍选优期望 AI 返回：

```json
{
  "task_type": "batch_pick",
  "best_capture_id": 123,
  "summary": "第 123 张主体更清晰、构图更稳定。",
  "score": 92
}
```

## 6. AI Provider 配置

```json
{
  "provider_code": "openai_main",
  "vendor_name": "openai",
  "provider_format": "openai_compatible",
  "display_name": "OpenAI Main",
  "api_base_url": "https://api.openai.com/v1/chat/completions",
  "api_key": "sk-...",
  "model_name": "gpt-4.1-mini",
  "enabled": true,
  "is_default": true,
  "notes": "production vision provider",
  "extra_config": {
    "timeout_seconds": 60,
    "max_tokens": 800,
    "temperature": 0.2
  }
}
```

当前后端只实现 `openai_compatible` 调用格式。`vendor_name=ollama` 时允许没有 `api_key`。
