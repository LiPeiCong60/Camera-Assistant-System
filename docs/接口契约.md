# Camera Assistant 接口契约

本文档按当前代码列出主要接口。更细的运行说明见 [技术架构与运行机制详解](./技术架构与运行机制详解.md)。

## 1. 通用约定

### backend envelope

`backend` 大多数接口返回：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

失败时通常由 FastAPI 返回 `detail`，移动端会把常见错误翻译成中文提示。

### device_runtime envelope

`device_runtime` 控制接口也使用：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

WebSocket 和 WebRTC video track 不使用 JSON envelope；WebRTC signaling 的 HTTP 响应仍使用 envelope。

### 地址

- 业务后端 base URL：`http://host:8000/api`
- 设备运行时 base URL：`http://host:8001`
- 管理后台前端：`http://host:5173`

真机联调必须使用局域网 IP。

## 3. 手机端到设备运行时

Base URL：`http://host:8001`

### 健康与状态

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/device/health` | 设备运行时健康状态。 |
| `GET` | `/api/device/status` | 当前 session、模式、云台、检测、模板、AI、overlay 状态。 |
| `PATCH` | `/api/device/config` | 更新 overlay 和 gesture 配置。 |

### 会话

`POST /api/device/session/open`

```json
{
  "session_code": "mobile-session",
  "stream_url": "mobile_push",
  "mirror_view": true,
  "start_mode": "MANUAL"
}
```

`stream_url` 可以是：

- `mobile_push`：手机推流，WebRTC 主链路和 fallback 都使用该值。
- `0`、`1`：OpenCV 本地摄像头编号。
- `rtsp://...`、`http://...`：OpenCV 可读取的视频流。

`POST /api/device/session/close`

```json
{
  "session_code": "mobile-session"
}
```

### WebRTC 主链路

`POST /api/device/webrtc/offer`

要求当前 session 的 `stream_url` 是 `mobile_push`。

请求：

```json
{
  "sdp": "...",
  "type": "offer"
}
```

响应：

```json
{
  "success": true,
  "message": "webrtc answer created",
  "data": {
    "sdp": "...",
    "type": "answer"
  }
}
```

语义：

- 手机端发送本机摄像头 video track。
- 设备端接收 track，转成 OpenCV BGR frame。
- 设备端把处理后的预览作为 video track 返回。
- signaling 是 HTTP；媒体流是 WebRTC。

### fallback 视频接口

`WS /api/device/stream/mobile-ws`

- 上行旧链路。
- 首条文本消息为配置，例如宽高、格式、旋转角。
- 后续二进制消息为 Android NV21 frame。
- 设备端解码后写入 `mobile_push_frame_store`。

`WS /api/device/preview-ws`

- 下行旧链路。
- 设备端持续发送 JPEG bytes。

`POST /api/device/stream/frame`

- 调试用单帧 JPEG 上传。
- multipart 字段名 `file`。

`GET /api/device/preview.jpg`

- 调试用单张 JPEG 预览。

## 5. 兼容说明

- WebRTC 是设备联动主路径。
- WebSocket/JPEG 接口仍是 fallback，不应删除。
- `backend` 不代理 WebRTC，也不保存设备 WebRTC 会话状态。
- `device_runtime` 是单会话模型，新 session 会关闭旧 session。
