# Camera Assistant 接口契约

## 1. 文档目的

本文档用于统一以下三方之间的接口字段与 JSON 结构：

- 手机端 `mobile_client`
- 树莓派运行时 `device_runtime`
- 业务后端 `backend`

目标：

- 统一字段命名
- 统一主链路对象结构
- 避免三端各自理解接口
- 为后续 FastAPI、Flutter、树莓派本地 API 开发提供同一份契约基线

本文档以 `docs/开发定稿.md` 为准，并与 `database/schema.sql` 对齐。

## 2. 总体约定

### 2.1 命名约定

- JSON 字段统一使用 `snake_case`
- 标识字段统一使用 `id` 或 `*_id`
- 编码类字段统一使用 `*_code`
- 时间字段统一使用 ISO 8601 字符串，例如：`2026-04-18T10:30:00Z`
- 状态字段统一使用 `status`

### 2.2 通用响应结构

成功响应：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

失败响应：

```json
{
  "success": false,
  "message": "invalid request",
  "error_code": "BAD_REQUEST"
}
```

分页响应：

```json
{
  "success": true,
  "message": "ok",
  "data": {
    "items": [],
    "page": 1,
    "page_size": 20,
    "total": 0
  }
}
```

### 2.3 通用状态值

建议首版使用以下状态枚举：

- 用户 `status`：`active` `inactive` `disabled`
- 设备 `status`：`offline` `online` `busy` `disabled`
- 订阅 `status`：`active` `expired` `cancelled` `paused`
- 模板 `status`：`active` `archived` `deleted`
- 拍摄会话 `status`：`opened` `closed` `cancelled`
- AI 任务 `status`：`pending` `running` `succeeded` `failed` `cancelled`

## 4. AI 结果 JSON 契约

该结构由后端返回给手机端，手机端可直接转发给树莓派执行。

```json
{
  "task_type": "background_lock",
  "recommended_pan_delta": -6.5,
  "recommended_tilt_delta": 3.0,
  "target_box_norm": [0.38, 0.18, 0.24, 0.66],
  "summary": "向左并略微抬高机位",
  "score": 84
}
```

字段说明：

- `task_type`：AI 任务类型，首版建议支持 `analyze_photo` `analyze_background` `batch_pick` `auto_angle` `background_lock`
- `recommended_pan_delta`：建议水平调整角度，单位为度
- `recommended_tilt_delta`：建议俯仰调整角度，单位为度
- `target_box_norm`：归一化站位框 `[x, y, w, h]`
- `summary`：自然语言建议
- `score`：AI 评分

## 8. 第一版实施说明

### 8.1 本文档优先级

若后续代码与本文档冲突，且未单独更新定稿文档，则以：

1. `docs/开发定稿.md`
2. 本文档

为准。

### 8.2 第一版允许的简化

- 上传类接口第一版可使用文件上传加表单字段，不强制一次性设计复杂对象
- `template_data`、`metadata`、`request_payload`、`response_payload` 第一版可先保留为 JSON
- 树莓派端第一版只要能解析本文档中的核心字段并执行，不要求一步到位实现全部扩展参数

### 8.3 第一版必须保持稳定的字段

以下字段一旦进入开发，不建议随意改名：

- `session_code`
- `device_code`
- `task_code`
- `task_type`
- `recommended_pan_delta`
- `recommended_tilt_delta`
- `target_box_norm`
- `capture_type`
- `template_data`
- `status`

## 2026-04-21 补充：`batch-pick` 真实多图语义

`POST /api/mobile/ai/batch-pick`

请求：
```json
{
  "session_id": 1,
  "capture_ids": [11, 12, 13]
}
```

成功时：
- `data.task.status = succeeded`
- `data.best_capture_id` 为 AI 选出的最佳图片
- backend 会将本次参与选优的抓拍里，最佳图写成 `is_ai_selected = true`

成功响应 `data` 示例：
```json
{
  "task": {
    "task_code": "AI_20260421_110512_123456",
    "task_type": "batch_pick",
    "status": "succeeded",
    "provider_name": "longcat_omni_primary",
    "result_summary": "第 12 张在主体姿态、构图和清晰度上整体最佳。",
    "result_score": 94
  },
  "best_capture_id": 12
}
```

失败时：
- `data.task.status = failed`
- `data.best_capture_id = null`
- 手机端不应把失败结果当成选优成功处理

## 2026-04-21 补充：管理后台推荐默认模板

### 新增字段

`Template` 结构补充：
- `is_recommended_default`：是否为后台维护的推荐默认模板
- `recommended_sort_order`：推荐模板在手机端列表中的排序值，数值越小越靠前

### 管理后台接口

`GET /api/admin/templates/recommended`

说明：
- 返回当前所有未删除的推荐默认模板
- 仅管理员可访问

`POST /api/admin/templates/recommended`

请求示例：

```json
{
  "name": "半身站姿推荐模板",
  "template_type": "pose",
  "source_image_url": null,
  "preview_image_url": null,
  "recommended_sort_order": 10,
  "status": "active",
  "template_data": {
    "bbox_norm": [0.3, 0.12, 0.38, 0.72],
    "pose_points": {
      "head": [0.49, 0.16],
      "left_shoulder": [0.43, 0.26],
      "right_shoulder": [0.55, 0.26]
    }
  }
}
```

`PUT /api/admin/templates/recommended/{template_id}`

说明：
- 请求体与创建接口一致
- 用于编辑推荐默认模板内容、状态和排序

`DELETE /api/admin/templates/recommended/{template_id}`

说明：
- 第一版按软删除处理
- 删除后手机端模板列表将不再返回该推荐模板

### 手机端模板列表补充约定

`GET /api/mobile/templates`

返回规则补充：
- 同时返回“当前用户自己的模板”和“后台推荐默认模板”
- 推荐默认模板按 `recommended_sort_order` 升序排在前面
- 用户自己的模板仍按创建顺序倒序返回

补充说明：
- 手机端允许选择推荐默认模板参与拍摄会话和设备联动
- 手机端仍只允许删除“当前登录用户自己创建的模板”
- 当 `is_recommended_default = true` 时，应视为后台维护模板，手机端不负责删除
