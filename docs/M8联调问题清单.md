# M8 联调问题清单

## 已修复问题

### 1. `capture_sessions.mode` 数据库约束与实际业务模式不一致

现象：
- 手机端创建拍摄会话时，写入 `SMART_COMPOSE`
- PostgreSQL 约束只允许 `mobile_only`、`device_link`
- 导致创建会话直接失败

处理结果：
- 已修复
- 当前兼容：
  - `mobile_only`
  - `device_link`
  - `MANUAL`
  - `AUTO_TRACK`
  - `SMART_COMPOSE`

### 2. `captures.capture_type` 数据库约束与手机端实际写入值不一致

现象：
- 手机端抓拍写库使用 `photo`
- PostgreSQL 约束不接受 `photo`
- 导致写入抓拍记录失败

处理结果：
- 已修复
- 当前兼容：
  - `single`
  - `photo`
  - `burst`
  - `best`
  - `background`

### 3. 后台高频页面和路由存在中文乱码

现象：
- 管理端部分页面、路由标题、错误提示显示乱码

处理结果：
- 已修复
- 已清理高频可见区域和核心路由元信息

## 当前仍存在但不阻塞交付的问题

### 1. `device_runtime` 对本地文件流会持续打印重连日志

影响：
- 不影响功能
- 会影响联调日志观感

建议：
- 后续给文件源单独加“读完后停止”的逻辑

### 2. MediaPipe Tasks 在当前环境会 fallback 到 solutions API

影响：
- 不影响当前主链功能
- 日志中会持续出现 fallback 提示

建议：
- 后续再单独收口运行时模型路径和初始化逻辑

### 3. `admin_web` 构建仍有 bundle 体积警告

影响：
- 不影响当前功能和验收
- 可能影响后续首屏加载性能

建议：
- 后续考虑路由级拆包和手动分 chunk

### 4. 手机端和管理端仍建议做一次手工端到端验收

影响：
- 当前代码和接口联调已通过
- 但模拟器/真机的最终交互体验仍建议手工确认

建议：
- 按联调验收清单逐项点击确认

### 5. 外部 AI 厂商网络调用仍建议做一次手工验证

影响：
- 本轮自动化联调已验证：
  - 上传链路
  - 任务写库
  - 结果结构化
  - 历史写回
- 但 provider 方法在脚本中做了打桩，未直接出网调用真实厂商

建议：
- 使用真实可用的 API Key，在当前环境中手工跑一次：
  - `analyze-photo`
  - `analyze-background`
  - `batch-pick`
- 确认厂商连通性、超时、返回格式与当前解析逻辑一致

## 当前结论

- 目前没有发现新的“阻塞交付级”问题
