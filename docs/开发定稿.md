# Camera Assistant 开发定稿

## 1. 项目目标

本项目是一套面向手机拍摄场景的智能拍摄辅助系统，包含手机端、树莓派端、云端 AI 与管理后台四部分。

项目目标分为两种运行模式：

1. 手机独立模式
- 手机端独立调用摄像头
- 本地显示人体框、骨架线、模板线
- 可在无树莓派情况下完成基础拍摄功能

2. 树莓派增强模式
- 手机端继续作为主交互入口
- 手机可向树莓派提供视频流与网络支持
- 树莓派负责实时检测、追踪、模板引导、云台控制
- 云端大模型负责图片分析、连拍选优、背景分析、找角度建议

项目当前目标不是一步做到完整商业化产品，而是优先完成可运行、可演示、可继续迭代的 60 分版本。

## 2. 最终产品定义

### 2.1 手机端职责

手机端是用户主入口，必须具备独立运行能力。

基础能力：
- 调用手机摄像头
- 实时画面预览
- 本地显示人体检测框
- 本地显示骨架线
- 本地显示模板线
- 单张拍照

增强能力：
- 与树莓派建立局域网联动
- 向树莓派提供视频流
- 手动控制云台
- 发起 AI 图片分析
- 发起 AI 连拍选优
- 接收云端 AI 返回结果
- 将 AI 返回的机位控制 JSON 转发给树莓派执行

### 2.2 树莓派端职责

树莓派不承担大模型推理，主要承担实时性强、设备控制强相关的任务。

核心能力：
- 接收手机视频流
- 本地实时人体检测/跟踪
- 控制云台舵机
- 执行手动控制命令
- 执行人物追踪
- 执行模板引导下的自动调机位
- 接收并执行 AI 返回的机位调整 JSON

### 2.3 云端 AI 职责

云端 AI 由第三方大模型服务商提供，不放在树莓派本地执行。

核心能力：
- 单张照片分析
- 背景图分析
- 模板图分析
- AI 连拍选优
- AI 自动找角度建议
- AI 背景分析并输出锁机位建议

云端 AI 的输出原则：
- 只输出建议与结构化 JSON
- 不直接控制树莓派硬件

### 2.4 管理端职责

管理端采用 Web 方案，定位为后台管理系统。

60 分版本优先实现：
- 用户管理
- 套餐/价格管理
- 拍摄记录管理
- AI 任务记录查看
- 基础统计面板

## 3. 技术栈定稿

### 3.1 手机端

- 技术选型：Flutter

选择原因：
- 摄像头能力成熟，适合移动端预览与拍照
- 适合在预览层上叠加人体框、骨架线、模板线
- 适合做双模式切换：独立模式 / 树莓派联动模式

### 3.2 管理端

- 技术选型：Vue 3 + Vite + Element Plus

选择原因：
- 开发后台速度快
- 表格、表单、筛选、弹窗等后台能力成熟
- 适合快速交付管理系统

### 3.3 业务后端

- 技术选型：Python + FastAPI

职责：
- 用户与套餐业务
- AI 任务中转
- 拍摄记录持久化
- 管理端 API
- 手机端业务 API

### 3.4 树莓派运行时

- 技术选型：Python
- 优先复用现有 `demo` 中的检测、追踪、模板与云台控制逻辑

### 3.5 数据库

- 技术选型：PostgreSQL

### 3.6 云端 AI

- 技术选型：第三方大模型 API
- 输出形式：JSON

## 4. 总体架构

系统分为四层：

1. 手机端
- 负责摄像头、实时预览、基础绘制、交互、拍照、上传、与树莓派联动

2. 树莓派端
- 负责实时检测、目标跟踪、模板引导、云台控制、执行机位 JSON

3. 业务后端
- 负责账号、套餐、模板、拍摄记录、AI 任务、管理后台 API

4. 云端 AI
- 负责非强实时视觉分析与评分

## 5. 两种运行模式

### 5.1 手机独立模式

特点：
- 不依赖树莓派
- 手机本地完成基础视觉叠加
- 适合“无云台”或“先独立拍摄”

能力范围：
- 摄像头预览
- 人体框
- 骨架线
- 模板线
- 拍照
- 单图上传 AI 分析
- 连拍上传云端选优

### 5.2 树莓派增强模式

特点：
- 手机仍为主入口
- 树莓派负责实时控制闭环

能力范围：
- 视频流发送到树莓派
- 树莓派实时跟踪人物
- 树莓派执行云台控制
- 树莓派执行模板引导自动调机位
- 树莓派接收 AI 返回 JSON 并调整机位

## 6. AI 功能定稿

### 6.1 AI 能力分类

基础 AI：
- 上传照片分析
- 上传背景图分析
- 上传模板图分析

高级 AI：
- 连拍统一上传后云端选优
- 自动找角度
- 背景分析并锁机位

### 6.2 AI 与树莓派的职责边界

最终原则：
- 云端 AI 负责分析与生成建议
- 树莓派负责执行舵机动作
- 手机负责拍图、上传、接收结果、转发给树莓派

### 6.3 AI 锁机位推荐链路

推荐链路如下：

1. 手机拍摄现场图或从预览中抽帧
2. 手机上传图片到业务后端
3. 业务后端请求云端 AI
4. 云端 AI 返回结构化 JSON
5. 业务后端把 JSON 返回给手机
6. 手机把 JSON 转发给树莓派
7. 树莓派解析 JSON 并执行舵机调整

主链路为：

`手机 -> 后端 -> 云端 AI -> 后端 -> 手机 -> 树莓派`

### 6.4 AI 输出 JSON 约定

示例：

```json
{
  "task_type": "background_lock",
  "recommended_pan_delta": -6.5,
  "recommended_tilt_delta": 3.0,
  "target_box_norm": [0.38, 0.18, 0.24, 0.66],
  "summary": "向左并略微抬高机位",
  "score": 84
}
```

说明：
- `recommended_pan_delta`：建议云台水平角度调整值
- `recommended_tilt_delta`：建议云台俯仰角度调整值
- `target_box_norm`：目标站位框，用于树莓派或手机端显示参考位置
- `summary`：自然语言建议
- `score`：评分

## 10. 数据库定稿

### 10.1 核心表

1. `users`
- 用户基本信息

2. `plans`
- 套餐信息

3. `user_subscriptions`
- 用户订阅关系

4. `devices`
- 用户树莓派设备

5. `templates`
- 用户模板数据

6. `capture_sessions`
- 一次拍摄会话

7. `captures`
- 实际拍摄图片记录

8. `ai_tasks`
- AI 任务与结果记录

### 10.2 表用途简述

`users`
- 管理普通用户与管理员

`plans`
- 管理服务定价与能力额度

`user_subscriptions`
- 管理用户当前订阅状态

`devices`
- 记录树莓派设备信息与在线状态

`templates`
- 存储用户模板图及结构化模板数据

`capture_sessions`
- 记录一次拍摄开始到结束的会话上下文

`captures`
- 记录单张图、连拍图、最佳图

`ai_tasks`
- 记录 AI 分析输入输出与状态

## 14. 当前结论

本项目最终定稿如下：

- 手机端：Flutter
- 管理端：Vue 3 + Vite + Element Plus
- 业务后端：Python + FastAPI
- 树莓派运行时：Python，优先复用 `demo`
- 数据库：PostgreSQL
- 云端 AI：第三方大模型 API

核心产品模式：
- 手机独立模式
- 树莓派增强模式

核心职责边界：
- 手机负责交互、拍照、上传、转发 AI 结果
- 树莓派负责实时检测和硬件执行
- 云端 AI 负责分析与建议
- 管理端负责后台管理
