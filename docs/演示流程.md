# 演示流程

本文给出演示或现场验收的推荐顺序。

## 准备

1. 后端启动并能访问 `GET /api/health`。
2. 管理后台启动并能登录。
3. 设备端启动并能访问 `GET /api/device/health`。
4. 手机端设置正确的业务后端地址和设备运行时地址。
5. 真机和树莓派在同一局域网，优先 5GHz Wi-Fi。

## 手机独立拍摄演示

1. 登录手机 App。
2. 进入拍摄页。
3. 选择或创建模板。
4. 拍摄照片。
5. 上传并进入历史。
6. 触发 AI 照片分析或背景分析。
7. 到历史页查看记录和 AI 结果。

## 管理后台演示

1. 登录管理后台。
2. 查看概览统计。
3. 查看用户和套餐。
4. 上传推荐模板。
5. 查看抓拍记录。
6. 查看 AI 任务。
7. 查看或修改 AI Provider 配置。

## 设备联动演示

1. 树莓派启动 `device_runtime`。
2. 手机设置设备运行时地址，例如 `http://192.168.1.30:8001`。
3. 设备联动页点击健康检查。
4. 打开 `mobile_push` 会话。
5. 启动手机推流，观察预览画面。
6. 打开人体骨骼/手部骨骼 overlay。
7. 切换跟随模式或手动控制云台。
8. 点击屏幕抓拍，确认 HUD 显示本地抓拍结果。
9. 打开手势抓拍，做有效手势。
10. 观察 3 秒倒计时提示。
11. 倒计时结束后查看设备抓拍列表。

## 现场卡顿排查

如果设备联动画面卡顿：

1. 确认树莓派使用 `DEVICE_RPI_PROFILE=performance`。
2. 关闭人脸网格，只保留必要的人体/手部检测。
3. 降低 `DEVICE_PREVIEW_SCALE` 和 `DEVICE_PREVIEW_FPS`。
4. 降低 `DEVICE_MAX_INFERENCE_SIDE` 和 `DEVICE_DETECTOR_FPS`。
5. 切换 5GHz Wi-Fi 或树莓派热点。
6. 避免后台同时跑高负载任务。

## 验收点

- 手机端能登录并加载首页。
- 手机独立拍照能写入后端历史。
- 后端 AI 任务能创建并显示结果或明确失败原因。
- 设备联动能打开会话并显示预览。
- 骨骼/手部 overlay 开关生效。
- 手势抓拍有提醒、有倒计时、有本地文件结果。
- 设备抓拍不要求出现在后端历史。
