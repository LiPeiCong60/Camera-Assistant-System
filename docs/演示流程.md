# 演示流程

## 目标

用于向项目验收方或团队成员演示：

- 手机端独立拍照链路
- 真实 AI 分析链路
- 设备联动链路
- 管理端查看与配置链路

## 演示前准备

1. 启动 PostgreSQL
2. 启动 `backend`
3. 启动 `device_runtime`
4. 启动 `admin_web`
5. 启动 `mobile_client`（模拟器或真机）

## 演示顺序

### 一、管理端准备

1. 登录管理端
2. 打开 `AI 配置`
3. 确认：
   - 已有可用 provider
   - 已启用
   - 已设为默认，或已被套餐绑定
4. 打开 `套餐管理`
5. 确认目标套餐已绑定 AI 配置
6. 如需演示设备绑定，先在 `设备列表` 中确认测试用户已绑定目标设备

### 二、手机端独立链路

1. 登录手机端
2. 进入基础拍照页
3. 选择模板
4. 拍照
5. 点击“上传并分析”
6. 展示：
   - 抓拍上传成功
   - AI 任务号
   - 分析摘要
   - 叠加层结果

### 三、手机端历史链路

1. 进入历史页
2. 查看：
   - 历史会话
   - 历史抓拍
3. 点击“AI 选优最近一组抓拍”
4. 展示最佳图已被标记

### 四、设备联动链路

1. 进入设备联动页
2. 确认当前测试方式：
   - 当前版本通常仍是手填设备地址
   - 如果已经预置测试地址，确认地址指向当前树莓派
3. 执行：
   - 健康检查
   - 打开会话
   - 手动控制
   - 切换 `follow-mode`
   - 模板下发
4. 执行 AI 链路：
   - `Apply Backend AI Angle`
   - `Apply Backend AI Lock`
5. 展示：
   - 设备状态刷新
   - `ai_lock_status.enabled=true`
   - 最近抓拍结果
6. 说明当前限制：
   - 设备选择还没有完全接到 backend 的用户设备绑定列表
   - 树莓派抓拍还没有自动回写到 backend AI 历史里
   - 演示 `Apply Backend AI ...` 时，更适合基于手机端已上传到 backend 的图片来展示

### 五、管理端回看

1. 回到管理端
2. 打开：
   - `拍摄记录`
   - `AI 任务`
   - `设备列表`
3. 展示：
   - 新增抓拍记录
   - AI 任务结果
   - 设备在线状态

## 演示重点提示

- 手机端上传的图片现在会先上传到 backend，再写抓拍记录
- AI 配置可由后台统一管理，并可被套餐绑定
- `analyze-photo`、`analyze-background`、`batch-pick` 已具备真实 provider 调用框架
- 设备端本地 API 可完成会话、控制、模板、AI 执行与抓拍触发
- 树莓派硬件路线当前统一为“TTL 总线舵机 + 独立电源”
- `PCA9685` 不再作为演示或部署方案
- 如果要做“设备抓拍 -> backend AI -> 再控舵机”的完整闭环演示，需要先补齐设备抓拍回传 backend 的链路
