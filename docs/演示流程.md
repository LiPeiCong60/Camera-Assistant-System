# 演示流程

本文档用于现场演示和验收。演示前请先按 [部署说明](./部署说明.md) 启动服务。

## 1. 演示目标

- 展示手机独立拍摄、模板和 AI 分析。
- 展示管理后台配置和数据回看。
- 展示设备联动已经使用 WebRTC 主视频流。
- 展示 WebRTC 失败时仍有 WebSocket/JPEG fallback。

## 2. 演示前准备

1. 启动 PostgreSQL。
2. 启动 `backend`：`uvicorn backend.app.main:app --host 0.0.0.0 --port 8000`。
3. 启动 `device_runtime`：`uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001`。
4. 启动 `admin_web`：`npm run dev`。
5. 手机安装最新 debug APK。
6. 手机和电脑/树莓派连接同一局域网。
7. 在手机设置页填写：
   - 业务后台地址：`http://电脑IP:8000/api`
   - 设备运行时地址：`http://设备IP:8001`
8. 手机浏览器访问 `http://设备IP:8001/api/device/health`，确认可达。

## 3. 演示顺序

### 3.1 管理后台

1. 登录管理后台。
2. 查看用户、套餐、设备登记。
3. 查看推荐模板。
4. 查看 AI Provider 配置。
5. 说明管理后台只访问 `backend`，不直接连接设备运行时。

### 3.2 手机独立拍摄

1. 登录手机端。
2. 打开拍摄页。
3. 选择模板或使用默认模板。
4. 拍摄照片。
5. 上传并创建抓拍记录。
6. 触发照片分析或背景分析。
7. 到历史页查看记录和 AI 结果。

### 3.3 连拍选优

1. 在手机端连续拍摄多张。
2. 触发连拍选优。
3. 展示后端 AI 任务结果。
4. 管理后台打开 AI 任务列表回看。

### 3.4 设备联动 WebRTC

1. 打开设备联动页。
2. 确认设备运行时地址是局域网 IP。
3. 打开 `mobile_push` 会话。
4. 开启手机画面推送。
5. 观察状态显示 WebRTC 推流。
6. 观察预览区域显示设备端返回的处理后画面。
7. 切换 `MANUAL`、`AUTO_TRACK`、`SMART_COMPOSE`。
8. 发送云台方向、回中、跟随模式等控制命令。
9. 触发设备抓拍。

### 3.5 fallback 验证

如果需要演示 fallback，可临时让 WebRTC 初始化失败或观察异常情况下的自动切换。fallback 成功时：

- 上行走 `WS /api/device/stream/mobile-ws`。
- 下行走 `WS /api/device/preview-ws`。
- 状态会显示 WebSocket/JPEG fallback 相关信息。

## 4. 演示风险与兜底

| 风险 | 检查项 |
| --- | --- |
| 真机连不上设备 | 确认同一局域网、局域网 IP、防火墙、`--host 0.0.0.0`。 |
| WebRTC 无画面 | 确认摄像头权限、`aiortc`、当前 session 是 `mobile_push`。 |
| fallback 握手失败 | 确认设备端安装 `websockets` 并重启服务。 |
| AI 任务失败 | 确认 AI Provider 配置、API key、模型和网络。 |
| 云台不动 | 本机 mock 模式只模拟角度，树莓派需确认 TTL 串口、ID、供电和共地。 |
