## 当前总体判断

按当前代码与文档对齐后的状态：

- 主体模块已齐：`backend`、`device_runtime`、`mobile_client`、`admin_web`
- 主链可运行：登录、拍照上传、AI 分析、设备会话、设备控制、后台管理都已有实现
- 树莓派硬件路线已收口为“TTL 总线舵机 + 独立电源”
- 当前阶段不再是“从零搭骨架”，而是“真实联动补齐 + 文档收口 + 真机验收”

当前重点：

- 补齐移动端与 backend 的设备绑定闭环
- 补齐树莓派抓拍回传 backend AI 的闭环
- 在真实树莓派上完成总线舵机方向与 home 角校准
- 完成部署与演示准备

## 当前模块状态

### 1. backend

当前状态：主链已可用

已完成：

- FastAPI 骨架
- PostgreSQL 接入
- 核心数据模型：用户、套餐、订阅、模板、拍摄会话、抓拍记录、AI 任务、AI Provider 配置、设备
- mobile API
- admin API
- 登录鉴权
- 图片上传到 backend 本地静态目录
- `analyze-photo` / `analyze-background` / `batch-pick` 的真实 provider 调用框架
- AI Provider 多厂商、多模型、多密钥配置管理

当前特点：

- backend 已不再只是占位接口，而是主链业务中心
- 后台 AI 配置、套餐绑定、历史记录、上传链路都已成型
- 设备表里已经有 `control_base_url` 等字段

### 2. device_runtime

当前状态：本地设备 API 可运行，真实总线舵机入口已接入

已完成：

- `status`

- `session/close`
- `manual-move`
- `mode`
- `follow-mode`
- 模板 `import / list / select / delete`
- `apply-angle`
- `apply-lock`
- `capture/trigger`
- `preview.jpg`

当前特点：

- Windows 本地开发默认使用 `mock`

- `PCA9685` 方案已废弃，不再作为当前设备端舵机控制路线
- 首版预览采用“最新帧 JPEG 预览”，不是低延迟实时视频

## 最近已经收口的关键点

1. 正式图片上传链路
- 手机端先上传图片文件到 backend
- backend 落盘到本地静态目录
- 返回 `/uploads/...` URL
- 抓拍记录改为使用 backend 可访问地址

2. 真实 AI 调用框架
- backend 已具备真实 provider 调用层
- 支持 OpenAI 兼容格式
- `analyze-photo`
- `analyze-background`
- `batch-pick`

3. 设备端真实总线舵机入口
- `device_runtime` 不再固定写死 `MockServoDriver`
- 已支持按环境变量切换 `mock` / `ttl_bus`
- 已明确废弃 `PCA9685`

4. 移动端产品化整改
- 设备联动页布局重构
- 首版设备预览
- network 图片预览支持
- 首页与登录页文案收口
- 网络错误提示统一优化

## 已知限制与风险

1. 设备预览不是低延迟实时视频
- 当前是“最新帧图片预览”
- 能满足首版联动画面需求
- 但不是完整 RTSP/WebRTC 播放体验

2. 移动端还没有打通“后端设备绑定 -> 设备联动页选择设备”的闭环
- backend 虽然有设备表和 `control_base_url`
- 但移动端当前仍主要依赖手填设备地址
- 这意味着 `device_id` 和真实绑定设备信息还不能稳定回流到联动页

3. 树莓派抓拍还没有完全回到 backend AI 闭环
- 设备端抓拍目前主要保留在本地路径和联动页状态里
- 还没有形成“树莓派抓拍 -> backend Capture 记录 -> backend AI -> 再回设备执行”的完整闭环
- 这会让“使用最新 backend AI 结果”时可能不是刚刚那张树莓派抓拍图

4. 云台方向语义还需要真机校准
- `manual-move` 路径与命令模式里的 `up/down` 当前定义不完全一致
- 在真实舵机联调前，需要统一 tilt 正负方向并做一次 home 角校准

5. device_runtime 日志观感仍不完美
- 本地文件流可能打印重连日志
- MediaPipe 可能 fallback 到 solutions API
- 当前功能可用，但日志不够干净

6. admin_web 仍有 bundle 体积警告
- 当前不阻塞功能或验收
- 后续如继续扩展后台，可再做拆包优化

## 本地启动方式

### 1. 启动 backend

```powershell
cd "E:\BianChengWenJian\Camera Assistant"
.\.venv\Scripts\Activate.ps1
$env:DATABASE_URL="postgresql+psycopg://postgres:<your_password>@127.0.0.1:5432/camera_assistant"
python -m backend.main
```

### 2. 启动 device_runtime

```powershell
cd "E:\BianChengWenJian\Camera Assistant"
.\.venv\Scripts\Activate.ps1
uvicorn device_runtime.api.app:app --host 0.0.0.0 --port 8001
```

### 3. 启动 admin_web

```powershell
cd "E:\BianChengWenJian\Camera Assistant\admin_web"
npm run dev
```

### 4. 启动 mobile_client（Android 模拟器）

```powershell
cd "E:\BianChengWenJian\Camera Assistant\mobile_client"
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000/api --dart-define=DEVICE_API_BASE_URL=http://10.0.2.2:8001
```
