# 文档清理说明

## 清理原则

本轮文档按当前代码重新整理，目标是减少重复、避免阶段性旧结论继续误导后续开发。

## 当前保留文档

| 文档 | 用途 |
| --- | --- |
| `项目总览与架构说明.md` | 主文档，说明项目背景、目标、架构、模块逻辑、协同方式和关键实现。 |
| `接口契约.md` | 手机端、设备端、管理端接口契约。 |
| `部署说明.md` | 本地和树莓派部署启动说明。 |
| `AI照片分析链路说明.md` | AI 图片分析、Provider 选择、结果解析和设备协同细节。 |
| `演示流程.md` | 演示和验收顺序。 |
| `文档清理说明.md` | 记录文档取舍，方便后续维护。 |

## 已删除文档

| 文档 | 原因 |
| --- | --- |
| `开发定稿.md` | 阶段性设计稿，内容已合并进主架构文档。 |
| `开发进展记录.md` | 历史日志过长，容易掩盖当前状态。 |
| `照片分析链路说明.md` | 与 `AI照片分析链路说明.md` 重复。 |
| `项目介绍文档.md` | 已被主架构文档覆盖。 |
| `项目当前状态说明.md` | 已合并进 README、主架构文档和部署说明。 |
| `项目详细审查报告.md` | 审查结论已提炼进主架构文档，避免保留过长的一次性报告。 |

## 后续维护建议

新增功能时优先更新对应稳定文档：

- 改接口：更新 `接口契约.md`。
- 改启动方式或依赖：更新 `部署说明.md` 和相关模块 README。
- 改 AI 调用或结果结构：更新 `AI照片分析链路说明.md`。
- 改模块职责或主流程：更新 `项目总览与架构说明.md`。
